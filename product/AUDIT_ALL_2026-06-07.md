# MainCharacter — Full Product Audit (all user-facing surfaces)

**Date:** 2026-06-07
**Auditor:** Head of Product Audit
**Scope:** `main` as it stands today. Landing, the two audit funnels, paywall/waitlist/payment-confirmed, Lookmaxxing pillar + fork, the PWA (`/lookmax/*`), the free tools, dashboard/welcome/terms/privacy/admin. Factoring in PR #36 (site-wide opt-in light mode) and PR #37 (free-tools relocation).
**Rule honoured:** No rewrites. Every fix below is an add / remove / tweak. Locked copy (`data/orator-content.js`, landing hero, rank ladder) is not flagged for rewrite — only true violations are.

> One framing note up front: the **live funnel is `/lookmaxing` → start → quiz → capture → audit/:id**, gated by a **₹99/month** paywall inside `audit.html`. The older surfaces (`/audit`, `paywall.html`, `upgrade.html`, `welcome.html`, `fork.html`) still exist on disk and on routes, and most of them carry a **different, contradictory pricing story** (₹799 / ₹1,499 / ₹1,999). The single biggest theme in this audit is **pricing incoherence across live surfaces.**

## Details by surface

### 1. Brand landing — `landing.html` (served at `/`)
- **Voice:** Clean. No emoji except ◆; the two emoji in the voice-comparison table (`🎉` at line 1474) are **intentional and semantic** ("what other apps say") — not a violation.
- **Pricing:** Uses **₹99/month** consistently in the hero trust line (1235) and Aura++ reveal (1413). This matches the live funnel — good.
- **Routing:** All primary CTAs now go to `/lookmaxing` (not `/audit`). Auth-aware nav swaps to "My Account"/dashboard when `lookmax.token` present. The Orator card opens a waitlist modal. Coherent.
- **Findings:** #12 ("Three scores"), #13 (quest framing), #14 (support TODO). All P3 — copy nuance, not breakage.
- **Light mode:** Page is fully tokenised (`--obsidian`, `--ink`, `--gold` → all in the light palette). The white radial glows over ivory are subtle and acceptable. No literal-colour breaks found.

### 2. Audit funnels
- **Legacy `/audit` (`public/audit.html`)** and **`/audit-legacy` (`index.html`)**: both still routed; nothing links to `/audit` anymore. `audit.html` ends by handing to `/paywall?auditSessionToken=…` — i.e. into the **stale ₹799/₹1,499/₹1,999 paywall**, not the ₹99 funnel. This is a parallel, inconsistent path. → #5, #2.
- **Live `/lookmaxing/audit/:id` (`audit.html`)**: this is the real conversion surface and is in good shape — proper loading **skeletons**, an **error state with Retry**, share-the-score viral hook, blurred premium blocks, Razorpay ₹99/month recurring. Voice is on-brand, no exclamation marks. The one issue is the **sticky-CTA / theme-toggle collision** (#3) and the **post-pay handoff to fork** (#1).

### 3. Paywall / waitlist / payment-confirmed
- **`paywall.html`** (`/paywall` when public): well-built 3-card page, but **wrong product & price** for the current funnel (#2). A/B tag bucketing and audit-echo are nice but moot if the page is off-path.
- **`paywall-waitlist.html`** (`/paywall` while gated): on-brand, captures phone, audit-echo. Fine. "The Chamber opens this weekend" is time-bound copy — confirm it's still accurate.
- **`payment-confirmed.html`**: strong. Polls `/api/payment/status` (30s), has waiting / confirmed / error states, silent first-login exchange, PWA install. Voice clean (◆ only). Built for the Orator+Lookmaxxing world (renders Orator step rows), which is harmless but dead weight while only ₹99 Lookmaxxing is live.

### 4. Lookmaxxing pillar + funnel
- **`/lookmaxing` index**: solid. Hero, how-it-works, two pillars (Lookmaxxing live / Orator waitlist), **new tools section (PR #37)**, repeat CTA. Light mode OK (uses `--mc-*`, all remapped). Tools-section findings: #8 (duplicate Studio link), #9 (token drift). The relocation reads reasonably in narrative flow — pillars → "here are free tools" → repeat CTA — but the section is link-dense (9 cards + 2 buttons) and slightly breaks the page's restrained cadence; trimming the duplicate button (#8) helps.
- **`start.html`** (sign-in): single email/password + Google path, 18+ consent, Terms/Privacy links present. Good. Google button degrades gracefully when unconfigured.
- **`quiz.html` / `capture.html`**: gated on token, redirect cleanly to `/lookmaxing/start` on 401. Forward CTAs present. No dead-ends.
- **`fork.html`**: the problem child — #1, #4, #10. Stale ₹1,499 + "₹99 credited" + disabled "opens tomorrow" premium card. Reachable only post-purchase, where it's most damaging.
- **`audit-full.html` / `full.html`**: duplicate "resolved report" pages, both `Continue →` to fork (#1, #10).

### 5. PWA — `/lookmax/*`
- Fixed bottom nav on every page → **theme-toggle overlaps the right-most tab** (#3). Hardcoded dark `theme-color` (#11). Settings surfaces dead "Aura++" plan (#15). Otherwise the PWA (dashboard, mirror, protocol, hair, reveal, settings) is coherent, on-brand, with proper auth redirects. Light mode technically applies (specificity of `:root[data-theme]` beats `:root`), but a paid installed dark app probably shouldn't ship a light toggle at all — recommend hiding it here (#3/#11).

### 6. Free tools — `tools/*`, `/face`, `/studio`
- Strong SEO scaffolding: unique `<title>`, `<meta description>`, `<link rel=canonical>` per page. **But no sitemap/robots** to tie them together (#6).
- Light-mode partial break: `tools.css` uses un-remapped `--dim/--faint/--panel2/--good` (#7) → dark card sub-text and nested fields on ivory.
- `/studio` is a paid (token) feature linked from the public landing tools section; confirm it gates purchase correctly for logged-out visitors (out of pure-frontend scope, flag for backend check).

### 7. Dashboard / welcome / terms / privacy / admin
- **`welcome.html`**: dead Orator page, **off-brand DM Sans font**, stale prices. → #5.
- **`terms` / `privacy`**: present and routed; linked from sign-in and footers. No dead-ends observed.
- **`admin.html`** (root, legacy React-flavoured file) contains `🔥` emoji (lines 389/540) but is **not the served admin** — `public/admin.html` is. The root `admin.html` is dead; recommend deleting to avoid the brand-voice landmine resurfacing. The served `public/admin.html` is internal-only (semantic status colours allowed), not in public scope.
